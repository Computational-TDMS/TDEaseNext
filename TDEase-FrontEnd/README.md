# 当前项目:

类似于vscode, 我们的工作流文件会放在后端,  前端主要负责对其进行渲染

但是我们所生成的tsv文件如果涉及可视化, 那么就需要传回前端, 使用typescript来进行处理与渲染,

此外, 包括氨基酸序列的编辑以及重算, 也都交给我们的前端部分来进行简单的运算.

而本地或者云端则负责重度的数据处理,生成概要文件, 生成tsv文件


不过前端也支持不进行数据处理, 直接使用纯粹可视化的工作流来实现数据的基本查看流程!

# 明确需求:


工具的参数由项目目录中config中的json来进行注册 而非硬编码, 从而确保灵活性. 
json会将命令行的参数, 输入输出进行分区对应, 并且给出默认的参数, 参数范围以及说明., 因此我们的工具基本类只需要读取即可知道前端该如何渲染!

我们的节点会需要正确渲染其输入与输出的连接点, 而对于其配置, 则在用户点开后, 在侧边栏进行细节的编辑
